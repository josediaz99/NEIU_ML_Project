Init_Win_bytes_forward	Init_Win_bytes_backward
- there refer to the initial tcp window size advertised by each side of the connection (client -> server -> client)
- they indicate how much data each side is willing to recieve before requiring acknoledgement 
-! large or unusual values can sometimes correlate with abnormal traffic or poor connections

act_data_pkt_fwd
- represents how many packets sent in the forward direction actually contains meaningful data
- represents data from users without the noise
-? if act data represents the data without the noise what kind of data is being sent when a dos attack is happening? 

Subflow Bwd Bytes   Subflow Fwd Bytes 
- these capture the total number of bytes sent forward vs backward within the session or flow
- this gives a sens of traffic inbalance 
-! dos attacks often heav heavy outbound packet volume wit hvery little legitimate response traffic
